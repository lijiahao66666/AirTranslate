## AirTranslate 重构方案（最终版）

### 核心功能
- 上传 EPUB 格式书籍
- 选择翻译引擎（微软/谷歌/HY-MT1.5-7B）
- 后台自动翻译
- 下载双语 EPUB

### 翻译引擎
| 引擎 | 段落翻译 | 章节翻译 | 上下文 | 术语表 |
|------|---------|---------|--------|--------|
| Microsoft | ✅ | ❌ | ❌ | ❌ |
| Google | ✅ | ❌ | ❌ | ❌ |
| HY-MT1.5-7B | ✅ | ✅ | ✅ | ✅ |

### 上下文选项
- 无上下文：直接翻译
- 轻量：前1段
- 标准：前3段

### 章节模式
- 整章一次性输入
- 超过4000字符按3000字符分割
- 后续段使用前1段或前3段上下文

### 术语表
- 前端提供术语表管理页面
- 用户自定义术语对照
- 翻译时携带术语表参考

### 调度器
- 公平调度，多任务交替执行
- 避免大书籍独占资源

### 项目结构
```
AirTranslate/
├── backend/          # Python FastAPI
├── scf/              # SCF Web函数
└── frontend/         # Flutter
```

### 实施计划
1. Phase 1: 后端核心（2-3天）
2. Phase 2: SCF函数（1天）
3. Phase 3: Flutter前端（3-4天）
4. Phase 4: 集成测试（1-2天）
